# Placement 重构任务说明

## 目标
将主程序中的 placement（物品放置与排序）系统抽离为独立模块，集中管理 prefab 实例放置、实例创建/删除、预览落地提交以及场景物品基础排序触发入口，在不改变现有行为的前提下提升结构清晰度。

## 允许的改动
- 新增 `src/application/placement/placement.js`。
- 将 placement 相关入口逻辑集中到该文件。
- 必要时保留兼容包装函数。
- 在 `index.html` 中新增最小必要的脚本接线。
- 在 `docs/` 下补充文档。

## 禁止的改动
- 不修改光影、人物移动、碰撞系统职责。
- 不重写物品排序算法本身。
- 不重写 render 管线。
- 不修改 scene / prefab 数据结构。
- 不修改 UI 行为。
- 不引入 ES module 或第三方依赖。

## 实现说明
- 本轮以“抽离与集中”为主，不做整体重设计。
- 优先保持行为稳定。
- 若排序逻辑与 render/logic 高度耦合，则本轮只抽 placement 侧触发入口，不强行搬动算法主体。
- 为降低风险，可保留旧的全局函数名作为兼容入口。

## 验收标准
- 主程序可正常打开。
- prefab 列表正常。
- 放置模式正常。
- 物品可正常放置。
- 若项目原本支持，移动/删除流程仍正常。
- scene 保存/读取不受影响。
- Habbo 资源放置不受影响。
- 后续开发者能够明确在 `src/application/placement/placement.js` 中查找 placement 入口逻辑。
